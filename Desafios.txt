Versão 1 -
 Sistema bancário v1 (Um usário)
 Operação de depósito:
 O usuário deve informar o valor a ser depositado. O valor deve ser positivo.Todos depósitos devem ser armazenados em uma lista.
 Operação de saque:
 O usuário deve informar o valor a ser sacado. O valor deve ser positivo e menor ou igual ao saldo disponível. O usuário só pode realizar 3 saques por dia. Todos os saques devem ser armazenados em uma lista.
 Operação de extrato:
 O sistema deve exibir todas as operações de depósito e saque, bem como o saldo atual. Se não houver movimentações, exibir a mensagem "Não foram realizadas movimentações."
 O sistema deve exibir um menu de opções para o usuário escolher a operação desejada (sacar, depositar, extrato ou sair). O programa deve continuar rodando até que o usuário escolha a opção de sair. 
 Observações: 3 saques diários, limite de saque de R$500,00 por operação, depósito mínimo de R$1. Caso não tenha saldo suficiente para realizar um saque, exibir a mensagem "Saldo insuficiente." O sistema deve ser implementado utilizando funções para cada operação (sacar, depositar, extrato) e uma função principal para controlar o fluxo do programa.

Versão 2 - 
 Novas funcionalidades do sistema bancário
 Separaras funções (saques, depósitos, extrato) em funções distintas
 Criar duas nocas funcionalidades: cadastrar usuário e criar conta bancária
   Função saque:
       Deve receceber os argumentos apensar por nome (keywords only)
   Função depósito:
       Deve receber os argumentos apenas por posição (positional only)
   Função extrato:
       Deve receber os argumentos por posição e nome (positional or keyword)
   Novas funcionalidades:
   Cadastrar usuário:
       Deve armazenar os usuários em uma lista, cada usuário deve conter nome, CPF, data de nascimento e endereço
       Endereço é uma string com o formato "Rua, Número, Bairro, Cidade e Sigla Estado"
       Armazenar somente os numeros do CPF, ou seja, sem pontos e traços; e não podemos cadastrar 2 usuários com o mesmo CPF
   Criar conta corrente:
       Armazenar as contas em uma lista, cada conta deve conter o número da conta, agência, titular
       O número da conta deve ser gerado automaticamente, iniciando em 1 e incrementando a cada nova conta criada
       O número da agência deve ser fixo, por exemplo, "0001"
       O usuário pode ter mais de uma conta, mas cada conta pertence a somente um usuário

Versão 4  - Decorador, Iterador e Gerador
    Decorador de log:
        Seja aplicado a todas funções de transações (depósito, saque, criação de conta, etc).
        Esse decorador deve registrar(printar) a data e hora de cada transação e bem como tipo de transação
    Gerador de relatório:
        iterar sobre as transações de uma conta e retorne, uma a uma, as transações que foram realizadas. Deve tambem ter uma forma de filtrar as transações baseado em seu tipo (por exemplo, apenas saques ou apoenas depósitos)
    Iterador personalizado:
        ContaIterador que permita iterar sobre todas as contas do banco, retornando informações básocas de cada conta (número,saldo, atual, etc)

Versão 6 - Manipulando dados
    Modificar o atual decorador de log, que imprime informações no console, para que ele salve essas informações em um arquivo de log, possibilitando uma revisão mais fácil e uma análise mais detalhada das operações dos usuários.
    
    Requisitos:
        O decorador deve registrar o seguinte para cada chamada de função:
            - Data e horas atuais
            - Nome da função
            - Argumentos da função
            - Valor retornado pela função
            - O arquivo de log deve ser chamado log.txt
            - Se o arquivo de log.txt ja existir, os novos log devem ser adicionados ao final do arquivo.
            - Cada entrada de log deve estar em uma nova linha
